Программа по обработке сигналов.

Программа предназначена для обработки экспериментальных данных, записанных цифровыми осциллографами в форматах CSV (и подобные текстовые форматы: DAT, TXT), WFM (бинарный формат фирмы Tektronix).

Обозначения:
Кривая/сигнал 	- запись зависимости амплитуды сигнала датчика от времени.

Столбец 		- последовательность чисел, записанная в файл.

X-cтолбец 		- последовательность отсчетов по времени какого-либо записанного сигнала.

Y-столбец 		- последовательность отсчетов по амплитуде какого-либо записанного сигнала.

Выстрел 		- одно событие, записанное в ходе эксперимента на один или несколько 
                  осциллографов в виде сигналов с датчиков. ВАЖНО: один выстрел может быть 
                  записан несколькими файлами, в т.ч. в разных форматах.

Размер группы 	- количество файлов, относящихся к одному выстрелу.

Num-first 		- (number first сортировка) шаблон нумерации файлов, при котором в начале 
                  имени файла указывается номер выстрела, и только после него указывается 
                  номер осциллографа и/или канала. Отсортированные по алфавиту файлы образуют 
                  группы по номерам выстрелов, размер группы равен количеству файлов одного 
                  выстрела. Это шаблон нумерации по умолчанию для осциллографов фирм 
                  Tektronix, Rohde & Schwarz.

Ch-first 		- (channel first cортировка) шаблон нумерации файлов, при котором в начале 
                  имени файла указывается номер осциллографа и/или канала, и только после 
                  этого указывается номер выстрела. Это шаблон нумерации файлов по умолчанию 
                  для осциллографов фирмы LeCroy.

Единый массив 	- двумерный массив, в который записываются данные, считанные из всех файлов, 
				  принадлежащих одному выстрелу. Стобцы 1, 3, 5 и т.д. хранят отсчеты по 
				  времени для сигналов № 1, 2, 3 и т.д. соответственно.
				  Столбцы 2, 4, 6 и т.д. хранят отсчеты по амплитуде для сигналов № 1, 2, 3 
				  и т.д.
				  Количество значений (строк) для разных сигналов может отличатся.


Возможности программы:

1. Считывание данных из файлов разных форматов с разной длиной записи, объединение их в единый массив и сохранение в формате CSV.
Входные файлы текстового типа (CSV, TXT, DAT) должны содержать четное число столбцов: по два (X и Y соответственно) столбца на каждую кривую.
Исключение: формат записи R&S HMO3054, где отсчеты по времени записаны в первом столбце и едины для всех кривых, амплитуды которых записаны в остальных столбцах. При записи четного числа каналов (2, 4) получаем нечетное число столбцов, - в этом случае программа рассматривает первый столбец, как X-столбец, а остальные - как Y-столбцы. 
ВАЖНО: при записи сигналов остциллографом R&S HMO3054 cледует записывать ТОЛЬКО 1, 2 или 4 канала одним файлом. Если записать 3 канала в один файл, то данные из такого файла будут считанны некорректно (как два сигнала с разными X-cтолбцами, вместо трех сигналов с одним X-столбцом)!

При считывании данных выстрела сигналы добавляются в единый массив последовательно: первый канал осциллографа, затем второй и так канал за каналом, файл за файлом. Файлы считываются в том порядке, в котором их имена были введены пользователем через флаг -f (--input-files), либо в алфавитном порядке, если указана директория с файлами, размер группы и тип сортировки (шаблон нумерации файлов).

2. Умножение данных на коэффициенты и вычитание задержки. И множители и задержки задаются упорядоченным списком значений, по одному значению для каждого столбца единого массива данных. Данные модифицируются по формуле:
data * M - D
где M, D - соответствующие множитель и задержка, data - считанные из файла данные.
ВАЖНО: вычитание задерже происходит после умножения данных на коэффициенты. 
Для X-столбцов задержка является временем, на которое задержан соответствующий сигнал, для Y-столбцов "задержка" является простым вычитаемым.
